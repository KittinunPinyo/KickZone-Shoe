# เอกสารวิเคราะห์และออกแบบระบบ (Analysis & Design)
## โครงงาน: KickZone (คิ๊กโซน) - ระบบเว็บไซต์ร้านค้าออนไลน์สำหรับจัดจำหน่ายรองเท้า

## 1. การวิเคราะห์ความต้องการ (Requirements Analysis)

### 1.1 ความต้องการของผู้ใช้งาน (User Requirements)

ระบบมีผู้ใช้งานหลัก 2 กลุ่ม คือ ลูกค้า (Customer) และ ผู้ดูแลระบบ (Admin)

**ลูกค้า (Customer)**
* สมัครสมาชิก / เข้าสู่ระบบ
* ค้นหาและเลือกซื้อสินค้ารองเท้า
* เพิ่มสินค้าลงตะกร้า (Cart)
* สั่งซื้อสินค้า (Checkout)
* ดูประวัติการสั่งซื้อ

**ผู้ดูแลระบบ (Admin)**
* จัดการข้อมูลสินค้า (เพิ่ม/แก้ไข/ลบ)
* จัดการหมวดหมู่/แบรนด์สินค้า
* ดูรายงานยอดขายและคำสั่งซื้อ
* จัดการผู้ใช้งานในระบบ

### 1.2 ขอบเขตของระบบ (System Scope)

ระบบครอบคลุมฟังก์ชันหลัก 9 ส่วน ตามที่กำหนดในรายวิชา ได้แก่

1. การจัดการสมาชิก (Register / Login)
2. การจัดการข้อมูลสินค้า
3. การค้นหาและแสดงรายละเอียดสินค้า
4. ระบบตะกร้าสินค้า (Shopping Cart)
5. ระบบสั่งซื้อสินค้า (Order Management)
6. ระบบชำระเงิน (Simulation)
7. ระบบติดตามสถานะคำสั่งซื้อ
8. ระบบจัดการสินค้าและคำสั่งซื้อสำหรับผู้ดูแลระบบ
9. รายงาน / Dashboard สรุปข้อมูล

### 2.1. การวิเคราะห์ระบบ (System Analysis - Functional Requirements)

#### 2.2 ฟังก์ชันสำหรับ ผู้ใช้งาน (User / Customer)
ลูกค้าร้านรองเท้าที่เข้ามาใช้บริการเว็บไซต์ สามารถทำรายการต่างๆ ได้ดังนี้:
*   **สมัครสมาชิก / เข้าสู่ระบบ:** สำหรับลงทะเบียนเป็นลูกค้าใหม่และเข้าสู่ระบบเพื่อทำรายการซื้อ
*   **ค้นหาและดูสินค้า:** 
    *   ผู้ใช้สามารถค้นหาสินค้าได้ 2 รูปแบบ คือ *ค้นหาด้วยชื่อสินค้า* และ *ค้นหาด้วยรหัสสินค้า*
*   **ดูรายละเอียดสินค้า:** 
    *   **Include:** เมื่อเปิดดูรายละเอียด ระบบจะต้อง **แสดงสินค้าคงเหลือ** เสมอ
    *   **Extend:** ผู้ใช้มีสิทธิเลือกที่จะ **เพิ่มเข้ารายการโปรด** ได้ หากสนใจสินค้านั้น
*   **จัดการตะกร้าสินค้า:**
    *   **Extend:** สามารถ **แก้ไขจำนวนสินค้า** ที่อยู่ในตะกร้าได้
    *   **Extend:** สามารถ **ลบรายการสินค้าออกจากตะกร้า** ได้
*   **สั่งซื้อสินค้า:**
    *   **Include:** ในขั้นตอนการสั่งซื้อ ระบบบังคับให้ผู้ใช้ต้อง **ระบุที่อยู่จัดส่ง** และเลือก **ช่องทางการจัดส่ง**
*   **ชำระเงิน:**
    *   รองรับช่องทางการชำระเงินย่อย ได้แก่ **ชำระเงินผ่านคิวอาร์** (QR Code) และ **ชำระเงินผ่านพร้อมเพย์** (PromptPay)
*   **ดูประวัติคำสั่งซื้อ:** สามารถเรียกดูประวัติการสั่งซื้อที่ผ่านมาของตนเองได้
*   **ติดตามสถานะคำสั่งซื้อ:** สามารถตรวจสอบสถานะการจัดส่งของคำสั่งซื้อได้

#### 2.3 ฟังก์ชันสำหรับ แอดมิน (Admin)
ผู้ดูแลระบบมีหน้าที่จัดการข้อมูลหลังบ้าน (Back-office) ดังนี้:
*   **จัดการข้อมูลสินค้า:**
    *   **Extend:** สามารถดำเนินการ **เพิ่มสินค้า / ลบสินค้า** 
    *   **Extend:** สามารถ **แก้ไขรายละเอียดสินค้า** ที่มีอยู่แล้วได้
*   **จัดการคำสั่งซื้อ:**
    *   **Extend:** สามารถ **อัพเดตสถานะการส่ง** ของสินค้าแต่ละรายการได้
    *   **Extend:** สามารถ **กรองดูรายการคำสั่งซื้อ** เพื่อความสะดวกในการจัดการ
*   **จัดการผู้ใช้งาน:**
    *   **Extend:** สามารถ **แก้ไขบทบาท** (Role) หรือสิทธิการเข้าถึงของผู้ใช้งานแต่ละคนได้
*   **สรุปยอดขาย:** สามารถเรียกดูรายงานและสรุปยอดขายของร้านค้าได้
